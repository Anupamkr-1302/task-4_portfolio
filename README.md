# Portfolio Website — Task 4 DevOps Version Control

## Overview
This repository contains my personal portfolio website built with HTML, CSS, and a PDF resume.  
It is maintained using Git best practices as part of Task 4 requirements.

## Branching Strategy
- **main** — production-ready code
- **dev** — integration/testing branch
- **feature/*** — short-lived feature branches

## How to View
Clone the repository and open `index.html` in any browser.

## Deliverables
- Proper commits and commit messages
- main, dev, and feature branches
- Pull requests used for merging
- `.gitignore` to exclude unwanted files
- Tags for releases
- Documentation in `docs/task-log.md`
